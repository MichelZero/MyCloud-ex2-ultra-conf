SSH Persistente e Automação de Desligamento

Configuração de chaves persistentes no NAS e criação de atalhos inteligentes no Proxmox PVE-01.

01

No Proxmox (PVE-01)

Gere a chave criptográfica e realize o primeiro envio para o NAS.

ssh-keygen -t ed25519 ssh-copy-id sshd@192.168.0.10
02

No NAS: Backup da Chave

Criamos um diretório no volume de dados (disco físico) para armazenar a chave permanentemente.

mkdir -p /mnt/HD/HD_a2/.ssh_backup/ cp ~/.ssh/authorized_keys /mnt/HD/HD_a2/.ssh_backup/
03

Script de Restauração Automatizada

Este script garante que a chave seja reinjetada na RAM após cada reinicialização do NAS.

nano /mnt/HD/HD_a2/.ssh_backup/restore_ssh.sh

Conteúdo atualizado do arquivo restore_ssh.sh:

#!/bin/sh
# 1. Define o caminho do home do root dinamicamente (Ex: /home/root)
ROOT_HOME=$(eval echo "~root")

# 2. Recria o diretório .ssh na RAM e aplica permissões de segurança
mkdir -p $ROOT_HOME/.ssh
chmod 700 $ROOT_HOME/.ssh

# 3. Restaura a chave autorizada do HD para a RAM (Sistema Ativo)
cp /mnt/HD/HD_a2/.ssh_backup/authorized_keys $ROOT_HOME/.ssh/
chmod 600 $ROOT_HOME/.ssh/authorized_keys
chmod +x /mnt/HD/HD_a2/.ssh_backup/restore_ssh.sh
04

Persistência no Boot do NAS

Vinculamos o script ao sistema de inicialização para execução automática.

ln -s /mnt/HD/HD_a2/.ssh_backup/restore_ssh.sh /opt/etc/init.d/S99restore_ssh
05

Dica de Ouro: Atalho no Proxmox

Crie um comando simples no PVE-01 para desligar o NAS de forma segura (OS 5 Way).

nano ~/.bashrc

Adicione esta linha ao final do arquivo:

alias desligar-nas='ssh sshd@192.168.0.10 "/usr/local/modules/script/shutdown.sh"'
source ~/.bashrc
SISTEMA PRONTO!
Use o comando "desligar-nas" no PVE para shutdown seguro.
