SSH Persistente para Proxmox (PVE)

Aprenda a configurar chaves SSH que sobrevivem ao reboot no WD MyCloud EX2 Ultra.

01

No Proxmox (PVE)

Gere a chave e envie para o NAS pela primeira vez para garantir o acesso inicial.

ssh-keygen -t ed25519 ssh-copy-id sshd@192.168.0.10
02

No NAS: Backup da Chave

Como a pasta root é limpa no boot, salvamos a chave no volume de dados do HD.

mkdir -p /mnt/HD/HD_a2/.ssh_backup/ cp ~/.ssh/authorized_keys /mnt/HD/HD_a2/.ssh_backup/
03

Criando o Script de Restauração

Este script irá repor a chave na memória RAM toda vez que o sistema iniciar.

nano /mnt/HD/HD_a2/.ssh_backup/restore_ssh.sh

Conteúdo do arquivo restore_ssh.sh:

#!/bin/sh
# 1. Define o caminho do root (no seu caso /home/root)
ROOT_HOME=$(eval echo "~root")

# 2. Recria a pasta .ssh na RAM e define permissões
mkdir -p $ROOT_HOME/.ssh
chmod 700 $ROOT_HOME/.ssh

# 3. Copia a chave do HD para a RAM
cp /mnt/HD/HD_a2/.ssh_backup/authorized_keys $ROOT_HOME/.ssh/
chmod 600 $ROOT_HOME/.ssh/authorized_keys
chmod +x /mnt/HD/HD_a2/.ssh_backup/restore_ssh.sh
04

Ativando a Persistência

Linkamos o script ao sistema de inicialização do Entware para execução automática.

ln -s /mnt/HD/HD_a2/.ssh_backup/restore_ssh.sh /opt/etc/init.d/S99restore_ssh
CONEXÃO ESTABELECIDA!
ssh sshd@192.168.0.10 -> Login direto OK
